## Appendix: Technical Specifications Summary

**For reference when asked for details:**

### System Architecture
- **4-layer design:** Detection, Memory, Recovery, Automation
- **4 detector types:** Project Switch, Keyword, Entity Mention, Token Threshold
- **Priority-based orchestration:** Detectors run in priority order (1-4)
- **Token budget management:** 5,000 token session limit, 500 per trigger
- **Graceful degradation:** Works without MCP, caches entity data

### Testing & Quality
- **214 test cases** across 9 test modules
- **93% code coverage** (industry standard: 80%+)
- **Test types:** Unit, integration, edge cases, mocking
- **CI/CD-ready:** Pytest with coverage reporting

### Performance
- **<1ms trigger evaluation** per detector
- **Intelligent caching:** Entity names (5min TTL), query results (10min TTL)
- **Lazy loading:** Detectors only load when enabled
- **Async-ready:** Architecture supports async operations

### Production Features
- **Logging:** Rotating file handlers, configurable levels
- **Error handling:** Retry logic, timeout enforcement, graceful failures
- **State management:** Session persistence, checkpoint validation
- **Monitoring:** Stats commands, debugging modes, dry-run previews

### Integration Points
- **MCP memory server:** Claude's persistent knowledge graph
- **Git hooks:** SessionStart, SessionEnd, post-commit
- **Task scheduler:** Periodic safety net checkpoints
- **Claude Code:** Hook integration via .claude/settings.json

### Code Statistics
- **~18,000 lines of code** (production + tests + docs)
- **43 production Python files**
- **9 test files**
- **47+ commits** over 47-day timeline
- **41 commits since Nov 1** (active development period)
