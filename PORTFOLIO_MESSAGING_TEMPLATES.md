### Technical Interview Talking Points

**Architecture Decisions:**
```
Q: Why a 4-layer architecture?
A: "Separation of concerns. Detection logic shouldn't know about storage. Recovery
shouldn't depend on automation. Each layer has clear interfaces, allowing independent
evolution. For example, I can swap MCP for a different storage backend without touching
detection logic."

Q: Why priority-based orchestration?
A: "Multiple triggers can fire simultaneously. Priority ensures the most important
context loads first. Project switch (priority 1) beats keyword match (priority 2) because
switching projects requires broader context. Plus, it respects token budgetsâ€”higher
priority triggers consume budget first."

Q: How do you handle failures?
A: "Graceful degradation everywhere. MCP unavailable? Fall back to local cache. Cache
stale? Return partial results. Network timeout? Retry with exponential backoff. The
system always worksâ€”it just degrades gracefully when services fail."
```

**Testing Strategy:**
```
Q: Why 214 tests?
A: "Three categories: unit tests for individual detectors, integration tests for the
orchestration engine, and edge case tests for error conditions. 93% coverage catches
regressions. Each detector has 15-20 tests covering normal operation, edge cases, and
error scenarios. The engine has 38 tests for state management, budget enforcement, and
multi-detector coordination."

Q: How do you test AI integration?
A: "Mocking. The MCP client has clear interfaces. Tests mock responses: successful queries,
empty results, timeouts, connection failures. This lets me test all code paths without
depending on external services. I also have integration tests that run against a local
MCP instance for end-to-end validation."
```

**Performance:**
```
Q: How do you achieve <1ms trigger evaluation?
A: "Intelligent caching. Entity names cache for 5 minutes, query results for 10 minutes.
Lazy loadingâ€”detectors only initialize when enabled. Regex pre-compilation for keyword
matching. Short-circuit evaluationâ€”if a higher-priority detector triggers, lower-priority
detectors don't run. These add up to sub-millisecond latency."
```
